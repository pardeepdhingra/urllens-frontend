## Overview

The URL Audit feature allows users to:
- **Batch Mode**: Audit multiple URLs for scrapeability
- **Domain Mode**: Discover and audit URLs from a domain (sitemap, robots.txt, common paths)

### Key Capabilities
- HTTP status and redirect chain analysis
- JavaScript rendering requirement detection
- Bot protection mechanism identification
- Scrape likelihood scoring (0-100)
- Best entry point recommendations

### Recommendation Logic

```typescript
function getRecommendation(score: number, accessible: boolean, hasBotProtection: boolean): string {
  if (!accessible) return 'blocked';
  if (score >= 85) return 'best_entry_point';
  if (score >= 70) return 'good';
  if (score >= 50) return 'moderate';
  if (hasBotProtection) return 'blocked';
  return 'challenging';
}
```

### Score Interpretation Guide

| Score Range | Recommendation | Meaning |
|-------------|----------------|---------|
| 85-100 | Best Entry Point | Ideal for scraping - fast, no protection |
| 70-84 | Good | Generally accessible with minor obstacles |
| 50-69 | Moderate | May require additional handling |
| 30-49 | Challenging | Significant obstacles present |
| 0-29 | Blocked | Not recommended for scraping |

## API Reference

### POST /api/audit

#### Request Body (Batch Mode)
```json
{
  "mode": "batch",
  "urls": [
    "https://example.com",
    "https://example.com/about",
    "https://example.com/contact"
  ]
}
```

#### Request Body (Domain Mode)
```json
{
  "mode": "domain",
  "domain": "example.com"
}
```

#### Rate Limits
- 5 audits per minute per user
- Maximum 100 URLs per batch
- Maximum 5 domains per audit
- 10 second timeout per URL
- 5 concurrent requests

## Database Schema

### url_audit_sessions
```sql
CREATE TABLE url_audit_sessions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES auth.users(id),
  mode TEXT NOT NULL,           -- 'batch' | 'domain'
  domain TEXT,                  -- For domain mode
  total_urls INTEGER NOT NULL,
  completed_urls INTEGER DEFAULT 0,
  status TEXT DEFAULT 'pending', -- pending|discovering|testing|scoring|completed|failed
  error TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  completed_at TIMESTAMPTZ
);
```

### url_audit_results
```sql
CREATE TABLE url_audit_results (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  session_id UUID NOT NULL REFERENCES url_audit_sessions(id),
  user_id UUID NOT NULL REFERENCES auth.users(id),
  url TEXT NOT NULL,
  status_code INTEGER,
  final_url TEXT,
  scrape_score INTEGER,         -- 0-100
  requires_js BOOLEAN DEFAULT false,
  bot_protections JSONB DEFAULT '[]',
  redirects JSONB DEFAULT '[]',
  accessible BOOLEAN,
  recommendation TEXT,          -- best_entry_point|good|moderate|challenging|blocked
  blocked_reason TEXT,
  content_type TEXT,
  response_time_ms INTEGER,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

## Future Modifications Guide

### Adding New Bot Protection Detection

Edit `auditEngine.ts`:

```typescript
const BOT_PROTECTION_PATTERNS = [
  // Add new protection here
  { name: 'NewProtection', patterns: ['pattern1', 'pattern2'] },
  // ... existing patterns
];
```

### Modifying Scoring Weights

Edit `calculateScoreBreakdown()` in `auditEngine.ts`:

```typescript
// Example: Increase HTTP status weight from 40 to 50
if (input.status === 200) {
  breakdown.httpStatus = 50;  // Changed from 40
}

// Remember to update total possible score!
```

### Adding New Discovery Sources

Edit `domainDiscoveryEngine.ts`:

```typescript
// Add new discovery method
async function discoverFromNewSource(baseUrl: string): Promise<DiscoveredURL[]> {
  // Your implementation
}

// Call it in discoverDomainURLs()
const newSourceUrls = await discoverFromNewSource(baseUrl);
result.discoveredUrls.push(...newSourceUrls);
```

### Adding New Recommendation Categories

1. Update `types/audit.ts`:
```typescript
export type AuditRecommendation =
  | 'best_entry_point'
  | 'good'
  | 'moderate'
  | 'challenging'
  | 'blocked'
  | 'new_category';  // Add new category
```

2. Update `getRecommendation()` in `auditEngine.ts`
3. Update `RecommendationChip` in `AuditResults.tsx`

### Implementing Paywall

1. Check subscription status in API route:
```typescript
// In route.ts
const hasProSubscription = await checkUserSubscription(user.id);
if (!hasProSubscription) {
  return NextResponse.json(
    { success: false, error: 'Pro subscription required' },
    { status: 403 }
  );
}
```

2. Update UI to show upgrade prompt
